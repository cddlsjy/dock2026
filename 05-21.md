这样一来，核心逻辑就简化成了两个部分：**一个半透明的悬浮窗按钮栏，和一个简单的自启动服务**。

### 🧩 给AI的侧边栏任务指令

#### 1. 项目目标

开发一个Android App，实现以下功能：

- 在屏幕左侧边缘显示一个**一直可见的、轻薄的图标栏**，和“氢桌面”左侧Dock栏的效果一样。
- 无需任何滑动或点击开关，图标栏开机就常驻在屏幕上。
- 点击每个图标，能直接启动你指定的车机应用（如导航、音乐等）。
- App随车机启动自动运行，无需手动点击。

#### 2. 简化方案（不依赖SmartSwipe库）

- **不搞侧滑**：直接用一个`悬浮窗`的方式显示这个工具栏，UI布局就用`LinearLayout`竖直排列几个带图标的`ImageButton`。
- **不搞复杂依赖**：不使用SmartSwipe等第三方库，就用Android原生方案，避免集成和兼容性问题。

#### 3. 详细实现步骤

**a) 添加权限**
在`AndroidManifest.xml`中添加两个核心权限：

- `SYSTEM_ALERT_WINDOW`：显示悬浮窗的权限。
- `RECEIVE_BOOT_COMPLETED`：接收开机广播的权限。

**b) 实现悬浮窗服务**

- 创建一个继承自`Service`的类，名为`FloatingIconService`。
- 在它的`onCreate()`或`onStartCommand()`里，利用`WindowManager`来添加一个自定义的悬浮布局。
- 布局是一个水平或竖直排列的容器，里面放若干个按钮（根据你的需求数量来定）。

**c) 图标与点击事件**

- 在`FloatingIconService`中，为每个图标设置点击监听，然后用`Intent`启动对应的应用。参考代码：

```java
// 启动导航应用
Button navBtn = findViewById(R.id.nav_btn);
navBtn.setOnClickListener(v -> {
    Intent intent = getPackageManager().getLaunchIntentForPackage("你的应用1包名");
    if (intent != null) {
        intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
        startActivity(intent);
    }
});

// 启动音乐应用
Button musicBtn = findViewById(R.id.music_btn);
musicBtn.setOnClickListener(v -> {
    Intent intent = getPackageManager().getLaunchIntentForPackage("你的应用2包名");
    if (intent != null) {
        intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
        startActivity(intent);
    }
});
```

> （这里自行替换`你的应用1包名`和`你的应用2包名`）

**d) 设置悬浮窗参数**
设置`WindowManager.LayoutParams`，让悬浮窗居于屏幕**左侧中间位置**，宽度自适应内容，高度根据按钮数量决定。

- `type`：设置为`TYPE_APPLICATION_OVERLAY`（Android 8.0+通用）。
- `flags`：设置为`FLAG_NOT_FOCUSABLE`，让触摸事件能穿透到悬浮窗下方的应用。

**e) 开机自启**

- 创建`BootReceiver`类，监听`BOOT_COMPLETED`广播。
- 收到广播后，启动`FloatingIconService`即可。

**f) 主界面**
主`Activity`是一个空白页面，一启动就运行`FloatingIconService`，然后把自己最小化（`moveTaskToBack(true)`），这样开机后就不会有个碍眼的黑窗口挡住车机主界面了。

#### 4. 给AI的最终要求

- **不要**用SmartSwipe库，直接用Android原生`WindowManager`实现。
- 图标包先使用系统自带的`@android:drawable/ic_menu_*`等占位图，方便后续替换。
- 整个项目使用Java编写。
- 最终产出：一个能直接安装运行的APK文件，和一个可以用Android Studio打开的项目源码压缩包。
